Task:
Implement a fully functional floating developer console for an Android APK (Kotlin) that replicates the behavior, UI structure, and functionality of the provided React console component.

Important Requirement:
The Android implementation must reproduce the same UI layout, controls, appearance, and features as closely as possible to the original React console. The goal is to create an Android in-app developer console similar to Android Studio Logcat, but embedded inside the application as a floating console window.

---

1. Technology Requirements

Use the following stack:

- Language: Kotlin
- UI: Jetpack Compose (preferred) or Android View/XML
- Architecture: MVVM
- Storage: Room Database
- Logging: Logcat + internal app logs
- File export: HTML export
- Clipboard: Android ClipboardManager

The console must run inside the APK, not as a separate tool.

---

2. Floating Console Window

Create a draggable and resizable floating console window inside the app.

The console must support:

- Dragging anywhere on screen
- Resizing from the corner
- Maximizing to full screen
- Minimizing back to floating size
- Closing the console

The design should visually resemble the React console window:

Header bar + log viewer area + controls.

---

3. Console Header Controls

The header must include the following buttons with identical behavior to the React console:

1. Pause / Resume logs
2. Refresh logs
3. Error/Warning viewer
4. Copy logs
5. Save logs
6. View saved logs
7. Export logs
8. Server / App log toggle
9. Maximize / Minimize
10. Close console

Each button must replicate the same functionality as the React implementation.

---

4. Log Sources

The console must support two log sources:

1. App Logs

Logs generated from:

- Log.d
- Log.i
- Log.w
- Log.e

2. System Logcat

Read logs from Android logcat and display them in real time.

---

5. Log Viewer UI

The log viewer must display logs in a scrollable list similar to a console.

Each log entry must show:

- Entry number
- Log level badge
- Timestamp
- Source
- Log message
- Optional metadata

Log level colors:

ERROR → red
WARN → yellow
INFO → blue
DEBUG → gray

Use a RecyclerView or LazyColumn to display logs.

---

6. Log Management Features

The console must support the following features exactly like the React version.

Copy Logs

Provide multiple copy options:

- Copy all logs
- Copy selected logs
- Copy log range
- Copy individual log

Use Android ClipboardManager.

---

Log Selection Mode

User must be able to:

- Select multiple logs
- Select all logs
- Clear selection
- Copy selected logs

---

Save Logs

Allow users to save logs as collections.

Saved collections must store:

- collection name
- log entries
- timestamp
- entry count

Use Room Database for persistence.

---

Load Saved Logs

User must be able to open saved collections and view them in the console.

When loading saved logs:

- pause live logging
- display stored logs

---

Delete Saved Logs

Allow deletion of saved log collections.

---

Export Logs

Allow exporting logs to a formatted HTML file.

The exported HTML should resemble a console-style log viewer, including:

- timestamps
- log level badges
- message text
- metadata

---

7. Error and Warning Viewer

Create a separate dialog or screen that shows:

- all ERROR logs
- all WARN logs

Display counts of:

- total errors
- total warnings

Allow copying all error logs to clipboard.

---

8. Real-Time Log Streaming

Logs must update in real time.

Use:

- Kotlin Flow
- StateFlow
- LiveData

Limit stored logs to about 1000 entries to avoid memory issues.

---

9. Mobile Interaction

The console must support:

- touch dragging
- touch resizing
- fast scrolling
- scroll to top
- scroll to bottom

---

10. Performance Requirements

The console must be optimized for performance:

- limit logs stored in memory
- efficient RecyclerView updates
- background log processing
- avoid UI thread blocking

---

11. UI Appearance

The UI should visually resemble the provided React console:

- floating window style
- header bar
- icon buttons
- log cards
- colored badges
- dialogs for saving and viewing logs

It should look like a mobile version of a developer console / Logcat viewer.

---

12. Deliverables

Provide:

1. Kotlin source code
2. Console overlay UI
3. Log adapter
4. Room database setup
5. Export system
6. Clipboard utilities
7. Logcat reader
8. Example usage in an Activity

The result should be a complete reusable Android console component that can be embedded into any app.

---

Goal:
Create an advanced in-app debugging console for Android that replicates the functionality and structure of the provided React console component.
